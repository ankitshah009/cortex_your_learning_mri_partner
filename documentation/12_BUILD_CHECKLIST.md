# Cortex Build Checklist

## Before coding

- [ ] Confirm the single demo problem.
- [ ] Confirm the expected first divergence.
- [ ] Confirm one primary misconception and two alternatives.
- [ ] Connect Butterbase MCP.
- [ ] Create application and verify one database write.
- [ ] Configure model credentials.
- [ ] Create public deployment shell.

## Static product

- [ ] Problem card.
- [ ] Reasoning input.
- [ ] Analyze CTA and loading states.
- [ ] Reasoning graph with four node states.
- [ ] Diagnosis, confidence, and evidence card.
- [ ] Probe card and response input.
- [ ] Updated diagnosis state.
- [ ] Short lesson, visual, and transfer problem.
- [ ] Reset action.

## Backend and AI

- [ ] Problem config.
- [ ] Diagnostic output schema.
- [ ] Initial diagnosis prompt.
- [ ] Verification prompt.
- [ ] Schema validator.
- [ ] Deterministic post-processor.
- [ ] Retry logic.
- [ ] Golden fixture fallback.
- [ ] Butterbase persistence.

## Reliability

- [ ] Known misconception test.
- [ ] Correct response test.
- [ ] Unstated assumption test.
- [ ] Arithmetic error test.
- [ ] Probe confirm test.
- [ ] Probe reject test.
- [ ] Provider failure test.
- [ ] Clean-browser deployment test.
- [ ] Five consecutive demo runs.
